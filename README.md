一、项目概述
本项目基于 C 语言实现了一个可动态调整线程数量的线程池，支持任务的异步提交、线程池的动态扩缩容（管理者线程根据任务队列状态和工作线程忙碌情况调整存活线程数）、线程池资源的安全销毁等核心功能。线程池采用生产者 - 消费者模型，通过互斥锁和条件变量保证多线程环境下的同步与互斥，具备高并发场景下的任务处理能力。
二、项目结构
.
├── main.c          # 测试代码，包含任务函数定义和线程池使用示例
├── threadpool.h    # 线程池相关函数声明、结构体前向声明
├── threadpool.c    # 线程池核心逻辑实现（创建、销毁、任务添加、线程管理等）
├── Makefile        # 编译构建脚本
└── README.md       # 项目说明文档（本文档）
三、编译与运行
# 编译生成可执行文件app
make
# 执行可执行文件
./app
# 删除目标文件（*.o）和可执行文件app
make clean
